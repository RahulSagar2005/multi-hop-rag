● This is a MultiHop RAG (Retrieval-Augmented Generation) Knowledge Graph
  project. Here's what it does:

  Overview

  The project combines knowledge graphs with LLM-powered retrieval to answer
  questions with verifiable citations. It consists of two main applications:

  1. Flask Web App (app.py)

  - User authentication system (login/register)
  - Landing page, dashboard, and profile pages
  - SQLite database for user management

  2. Streamlit Knowledge Graph App (knowledge_graph_rag.py)

  The main RAG system with these features:

  Document Processing:
  - Uses Ollama (local LLM) to extract entities and relationships from documents
  - Entities are categorized (PERSON, ORGANIZATION, TECHNOLOGY, CONCEPT, etc.)
  - Stores everything in Neo4j graph database with full source provenance

  Multi-Hop Querying:
  - Searches the knowledge graph for relevant entities
  - Traverses 1-4 hops from seed entities to find connected information
  - Generates answers with inline citations linking back to source documents

  Key Technologies:
  - Neo4j - Graph database for storing entities/relationships
  - Ollama - Local LLM for entity extraction and answer generation
  - Streamlit - UI for the RAG system
  - Flask - Web frontend with user auth

  Sample use case: Ask "Who developed GraphRAG and what organization are they
  from?" - the system traverses the graph to find connections and provides cited
   answers.
